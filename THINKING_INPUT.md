# 销量预测优化 - Thinking模型输入

## 项目背景
- 电商SKU销量预测，使用Amazon Chronos-2时序模型
- 59个SKU，训练期2023-01-01至2025-09-30，测试期60天(2025-10-01至2025-11-29)
- 当前最佳准确率：70.5%

## 已尝试的优化方案

| 版本 | 准确率 | 方法 | 结论 |
|------|--------|------|------|
| V1 基线 | 66.7% | 3协变量，一次预测60天 | 基线 |
| V2 | 70.5% | +时间特征，+滚动预测(7天) | 有效 |
| V3 | 72.1% | +分层集成(硬编码) | 不可泛化 |
| **生产版** | **70.5%** | V2无硬编码 | 当前最优 |
| V4 多分位数 | 69.5% | 多分位数加权 | 无效 |
| V5 动态校正 | 69.7% | 残差校正 | 无效 |
| V6 滞后特征 | 62.8% | +lag特征+节假日 | 下降 |
| V7 生命周期 | 64-67% | 增长/衰退系数 | 下降 |
| V8 分层集成 | 66.5% | Chronos+LightGBM | LightGBM过拟合 |
| V9 简化分层 | 68.6% | 智能截断+中位数收缩 | 后处理破坏预测 |
| Croston | 46.1% | 间歇需求模型 | 不适用 |
| 多SKU联合 | 70.5% | 多SKU同时预测 | 无提升 |

## 关键发现

### 1. CV是准确率的核心预测因子
- CV与准确率相关系数：-0.68
- 稳定型(CV<1.2)：32个SKU，平均75.3%
- 波动型(CV≥1.2)：27个SKU，平均65.0%
- 间歇型(CV≥2.5)：8个SKU，平均52%

### 2. Chronos-2已学到波动模式
- 对波动型SKU的任何后处理都降低准确率
- 智能截断、中位数收缩、LightGBM集成均无效
- 模型本身已捕获趋势，额外校正导致双重校正

### 3. 数据泄露问题
- 未来协变量只能包含可提前规划的字段
- 可用：is_promo, discount_rate, ppc_fee, 时间特征
- 不可用：sessions, ppc_clicks, conversion_rate（未知）

### 4. 小样本限制
- 59 SKU × 60天 = 3540条测试记录
- LightGBM等ML模型严重过拟合
- 需要更多数据或更简单的方法

## 当前生产版方案

```
协变量设计：
- 历史(context)：quantity, sessions, ppc_clicks, ppc_ad_order_quantity, 
                 conversion_rate, is_promo, discount_rate, ppc_fee, 
                 day_of_week, is_weekend, month, qty_yoy
- 未来(future)：is_promo, discount_rate, ppc_fee, day_of_week, is_weekend, month

滚动预测：每7天更新context，用实际值滚动
```
